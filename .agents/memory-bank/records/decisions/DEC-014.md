# DEC-014 — Multi-agent memory-bank isolation

| Field | Value |
|---|---|
| Status | Accepted |
| Date | 2026-06-21 |
| Modules | all (context layer) |

## Context

The legacy flat memory-bank (`activeContext.md`, `progress.md`, `decisions.md`) funneled every
session's volatile state into three shared files that were rewritten or top-appended each session.
Two parallel agents (in separate worktrees) editing the same file at the same line regions
produced guaranteed merge conflicts at PR time, and silent semantic conflicts (two agents minting
the same `DEC-014`).

## Decision

Partition the memory-bank so that **parallel agents working in separate worktrees never edit the
same file**:

1. **Reference files** (`projectbrief.md`, `systemPatterns.md`, `techContext.md`, prose of
   `productContext.md`) — hand-edited, low-frequency, low conflict risk.
2. **Generated index files** (`activeContext.md`, `progress.md`, `decisions.md`, `productContext.md`
   traceability tables, `plans/00-index.md`) — regenerated deterministically by
   `scripts/sync-memory-index.sh` from registries + records. Two agents produce identical output →
   no merge conflict. `--check` is a CI gate.
3. **Append-only JSONL registries** (`registry/*.jsonl`) — stable IDs (`REQ-###`, `NFR-###`,
   `SCN-###`, `TEST-###`, `DEC-###`, `RISK-###`, `TASK-###`). One object per line. Conflicts
   collapse to last-line and are trivially resolvable by re-reading `max+1`.
4. **Per-record files** (`records/progress/M{NN}.md`, `records/active/M{NN}.md`,
   `records/decisions/DEC-###.md`) — one file per record. Two agents completing different milestones
   create distinct files → zero merge conflict.
5. **Module locks** (`locks/<module>.md`) — record which agent/branch owns a module before editing
   coupled file pairs. Turn silent semantic breakages into textual conflicts.
6. **Worktree scratchpads** (`worktrees/<branch-slug>/`, git-ignored) — per-agent "Current Focus"
   drafts that never merge to main.

## Consequences

- `activeContext.md` no longer hand-edited; replaced by `records/active/M{NN}.md` per active
  milestone + generated aggregate.
- `progress.md` and `decisions.md` are generated from `records/` + `registry/`; never hand-edited.
- To mint a `DEC-###` / `REQ-###` / etc.: read the registry, take `max+1`, append one line, then
  create the corresponding `records/**` file. Never edit an existing registry line.
- Before editing coupled file pairs (e.g. a prompt template and its parser), acquire
  `locks/<module>.md`.
- CI must run `scripts/sync-memory-index.sh --check` to forbid hand-edited index files.

## Migration

Legacy `DEC-001`..`DEC-013` were migrated from `decisions.md` into `registry/dec.jsonl` +
`records/decisions/DEC-00N.md`. Legacy IDs are immutable. The legacy flat files were deleted.
